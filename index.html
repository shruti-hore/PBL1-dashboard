Overview

Problem & Motivation

Problem Statement

Remaining Useful Life (RUL) models perform well in single-condition settings like FD001. However, performance collapses in FD004 — featuring 6 operating conditions and 2 fault modes — because identical sensor readings carry different physical meanings across conditions.

Why Existing Models Fail

  • Sensor Ambiguity: Models confuse condition changes with degradation signals.
  • Domain Shift: Training distributions differ from test in complex altitude/speed regimes.
  • Flat Predictions: LSTM/GRU often output constant predictions (~125 cycles) ignoring sensor fluctuations.

Approach

DA-TCN Pipeline

A 4-stage pipeline designed to address domain shift and improve RUL accuracy.

01

Preprocessing

  • Remove low-variance sensors
  • K-Means clustering (6 groups)
  • Per-condition normalization
02

Delta Augmentation

  • Concatenate raw features [X]
  • First-order differences [ΔX]
  • Higher-order rate-of-change
03

TCN Architecture

  • 4 Dilated blocks (d=1,2,4,8)
  • Self-Attention pooling
  • Residual connections
04

Loss Function

  • Adaptive Monotonic Loss
  • Penalizes late predictions
  • Safety-aware optimization

The DA-TCN model successfully handles domain shift in the NASA FD004 dataset, delivering a robust, physically consistent, and safety-aware solution for turbofan engine maintenance.

Domain Shift Resolved — Per-condition normalization + clustering eliminates sensor ambiguity.
Superior Accuracy — RMSE 14.39 vs 34.66 (LSTM) and 51.47 (RF) on FD004.
Safety-First Design — Adaptive Monotonic Loss ensures conservative predictions.
Lightweight Architecture — ~68K parameters, deployable on edge hardware.
